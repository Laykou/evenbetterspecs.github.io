Factories, not fixtures

Factories are more flexible and easy to work with.

{% highlight ruby %} def name "#{first_name} #{last_name}" end {% endhighlight %}
{% highlight ruby %} it 'returns the full name' do user = create(:user, fist_name: 'Ayrton', last_name: 'Senna') expect(user.name).to eq('Ayrton Senna') end {% endhighlight %}

If your code doesn't hit the database, prefer build_stubbed over create since the former will make your tests faster ðŸ”¥
